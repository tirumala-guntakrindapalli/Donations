# Configuration Modes Guide

## How Configuration Works

Your application now properly supports **two modes** with different configuration sources:

### 🧪 Mode 1: Local Development (TEST_MODE = true)
**When**: Running locally with `./start.sh`  
**Config Source**: `dashboard-config.js` (local file, gitignored)  
**How it works**:
```javascript
const DASHBOARD_CONFIG = {
    TEST_MODE: true,          // ✅ Use local files
    USE_GITHUB: false,        // ❌ Don't use GitHub API
    ADMIN_PASSWORD_HASH: '...', // Your local hash
    GITHUB_TOKEN: '',         // Not needed
    // ... other config
};
```

**Data Loading**: Loads from `data/donations-YEAR.json` files directly  
**Saves**: In-memory only (not persistent unless you manually save JSON)

---

### 🚀 Mode 2: Production/GitHub Pages (TEST_MODE = false)
**When**: Deployed on GitHub Pages  
**Config Source**: Generated by GitHub Actions from **GitHub Secrets**  
**How it works**:

#### GitHub Actions Workflow (.github/workflows/deploy.yml)
```yaml
- name: Create dashboard-config.js from secrets
  run: |
    cat > dashboard-config.js << 'EOF'
    const DASHBOARD_CONFIG = {
        TEST_MODE: false,                              // ✅ Production mode
        USE_GITHUB: true,                              // ✅ Use GitHub API
        ADMIN_PASSWORD_HASH: '${{ secrets.ADMIN_PASSWORD_HASH }}', // From secret
        GITHUB_TOKEN: '${{ secrets.GITHUB_TOKEN_OPTIONAL }}',       // From secret
        GITHUB_REPO: '${{ secrets.GITHUB_REPO || github.event.repository.name }}',
        GITHUB_OWNER: '${{ secrets.GITHUB_OWNER || github.repository_owner }}',
        // ... other config
    };
    EOF
```

**Data Loading**: Fetches from GitHub raw files  
**Saves**: Uses GitHub API to commit changes (if token provided)

## Code Implementation

### In simple-dashboard.js

```javascript
// Load Data from GitHub or Local
async function loadDataFromGitHub() {
    // Get configuration
    const testMode = DASHBOARD_CONFIG.TEST_MODE;
    const currentYear = DASHBOARD_CONFIG.currentYear;
    
    if (testMode) {
        // LOCAL MODE: Load from local file
        console.log(`🧪 TEST MODE: Loading local data for year ${currentYear}...`);
        const dataPath = `data/donations-${currentYear}.json`;
        const response = await fetch(dataPath + '?t=' + new Date().getTime());
        currentData = await response.json();
    } else {
        // PRODUCTION MODE: Load from GitHub
        const url = `https://raw.githubusercontent.com/${DASHBOARD_CONFIG.GITHUB_OWNER}/${DASHBOARD_CONFIG.GITHUB_REPO}/${DASHBOARD_CONFIG.GITHUB_BRANCH}/data/donations-${currentYear}.json`;
        const response = await fetch(url + '?t=' + new Date().getTime());
        currentData = await response.json();
    }
    
    processData();
}

// Save Data to GitHub or Local
async function saveDataToGitHub() {
    if (!isAdmin) return;
    
    const testMode = DASHBOARD_CONFIG.TEST_MODE;
    
    if (testMode) {
        // LOCAL MODE: Update in memory only
        console.log('🧪 TEST MODE: Data saved in memory (temporary)');
        currentData.lastUpdated = new Date().toISOString();
        processData();
        showSuccess('✅ Changes saved temporarily! (Lost on refresh - TEST MODE)');
    } else {
        // PRODUCTION MODE: Save to GitHub via API
        const sha = await getFileSHA();
        const content = btoa(JSON.stringify(currentData, null, 2));
        
        const url = `https://api.github.com/repos/${DASHBOARD_CONFIG.GITHUB_OWNER}/${DASHBOARD_CONFIG.GITHUB_REPO}/contents/data/donations-${DASHBOARD_CONFIG.currentYear}.json`;
        
        const response = await fetch(url, {
            method: 'PUT',
            headers: {
                'Authorization': `token ${DASHBOARD_CONFIG.GITHUB_TOKEN}`,
                'Content-Type': 'application/json'
            },
            body: JSON.stringify({
                message: `Update data - ${new Date().toLocaleString()}`,
                content: content,
                sha: sha,
                branch: DASHBOARD_CONFIG.GITHUB_BRANCH
            })
        });
        
        if (response.ok) {
            showSuccess('✅ Data saved successfully to GitHub!');
        }
    }
}
```

## GitHub Secrets Setup

For production deployment, configure these secrets in GitHub:

**Repository → Settings → Secrets and variables → Actions → New repository secret**

### Required Secret:
- **Name**: `ADMIN_PASSWORD_HASH`
- **Value**: Your SHA-256 password hash
  ```bash
  echo -n "your-password" | shasum -a 256
  ```

### Optional Secrets (for GitHub API integration):
- **Name**: `GITHUB_TOKEN_OPTIONAL`
- **Value**: GitHub Personal Access Token (for saving data via API)

- **Name**: `GITHUB_REPO`  
- **Value**: Your repository name (auto-detected if not set)

- **Name**: `GITHUB_OWNER`
- **Value**: Your GitHub username (auto-detected if not set)

## Summary

| Feature | Local Mode (TEST_MODE=true) | Production Mode (TEST_MODE=false) |
|---------|----------------------------|----------------------------------|
| **Config File** | dashboard-config.js (local) | dashboard-config.js (generated) |
| **Config Source** | Your local file | GitHub Secrets |
| **Data Loading** | Local files (data/*.json) | GitHub raw CDN |
| **Data Saving** | In-memory (temporary) | GitHub API (permanent) |
| **Password** | Your local hash | Hash from secret |
| **Git Commits** | Config is gitignored | Config generated at build |
| **Use Case** | Development & testing | Production hosting |

**✅ Both modes now work correctly with proper configuration sources!**
